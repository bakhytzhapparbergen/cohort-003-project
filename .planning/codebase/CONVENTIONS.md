# CONVENTIONS.md — Code Style & Patterns

## Language & Runtime
- TypeScript throughout — strict mode enabled
- ES modules (`.ts`, `.tsx`)
- Node.js 20+ (Vite/React Router SSR)

## Formatting
- Prettier with default config (`.prettierrc` minimal)
- 2-space indentation
- Single quotes for strings in most files

## Naming Conventions

| Entity | Pattern | Example |
|---|---|---|
| Functions | camelCase | `getLessonById` |
| Types/Interfaces | PascalCase | `ParseResult<T>` |
| Enums | PascalCase with PascalCase members | `UserRole.Student` |
| Files (routes) | dot-notation kebab | `courses.$slug.tsx` |
| Files (services) | camelCase + `Service` | `lessonService.ts` |
| Files (components) | kebab-case | `course-image.tsx` |
| DB columns | camelCase (drizzle maps to snake_case) | `instructorId` |
| Constants | camelCase or UPPER_SNAKE if truly constant | — |

## Multi-Parameter Functions (CLAUDE.md Rule)
When a function has **2+ parameters of the same type**, use an object parameter:

```typescript
// BAD
const addUserToPost = (userId: string, postId: string) => {};

// GOOD
const addUserToPost = (opts: { userId: string; postId: string }) => {};
```

Single-type or clearly ordered parameters use positional form:
```typescript
export function getLessonById(id: number) { ... }
export function createLesson(moduleId: number, title: string, ...) { ... }
```

## Service Pattern
Services are plain TypeScript modules — no classes, no DI:
```typescript
import { eq } from "drizzle-orm";
import { db } from "~/db";
import { lessons } from "~/db/schema";

export function getLessonById(id: number) {
  return db.select().from(lessons).where(eq(lessons.id, id)).get();
}
```
- Return raw drizzle results (no DTO transformation)
- Use `drizzle-orm` operators (`eq`, `and`, `sql`, etc.)
- `.get()` for single row, `.all()` for multiple

## Route Module Pattern
React Router v7 with typed loaders/actions via `.react-router/types/`:

```typescript
import type { Route } from "./+types/my-route";

export async function loader({ request, params }: Route.LoaderArgs) {
  const session = await getSession(request);
  // ... fetch data
  return { data };
}

export async function action({ request }: Route.ActionArgs) {
  const formData = await request.formData();
  const result = parseFormData(formData, MySchema);
  if (!result.success) return data({ errors: result.errors }, { status: 400 });
  // ... perform action
  return redirect("/somewhere");
}

export default function MyRoute({ loaderData }: Route.ComponentProps) {
  return <div>{loaderData.data}</div>;
}
```

## Form Validation Pattern
Uses Valibot with `parseFormData` helper from `~/lib/validation`:

```typescript
import * as v from "valibot";
import { parseFormData } from "~/lib/validation";

const MySchema = v.object({
  title: v.pipe(v.string(), v.minLength(1)),
  price: v.pipe(v.string(), v.transform(Number)),
});

const result = parseFormData(formData, MySchema);
if (!result.success) return data({ errors: result.errors }, { status: 400 });
```

## Error Handling
- Route loaders/actions: throw `data({}, { status: N })` or `redirect()`
- Services: return `null` / falsy for not-found; let callers handle
- Validation: return `{ success: false, errors }` — never throw from services
- No global error boundaries beyond React Router defaults

## Session Pattern
iron-session cookie-based auth via `~/lib/session.ts`:
```typescript
const session = await getSession(request);
if (!session.userId) throw redirect("/login");
```

## Imports
- Path alias `~` maps to `app/` (`tsconfig.json` `paths`)
- React Router utilities from `react-router` (not `@remix-run/react`)
- UI components from `~/components/ui/<name>`
- Services imported directly: `import { getLessonById } from "~/services/lessonService"`

## Comments
- Section dividers use `// ─── Title ───` pattern in some files
- No inline documentation for obvious logic
- Complex helpers have JSDoc blocks (e.g., `parseFormData`, `createTestDb`)

## CSS / Styling
- Tailwind CSS utility classes
- `cn()` helper from `~/lib/utils` for conditional classes (clsx + tailwind-merge)
- shadcn/ui components for primitives

## TypeScript Strictness
- `strict: true` in tsconfig
- Avoid `any` — use specific types or `unknown`
- Route types generated by React Router in `.react-router/types/`
