# External Integrations

**Analysis Date:** 2026-04-17

## APIs & External Services

**Geolocation/IP Resolution:**
- IP-API (`ip-api.com`) - Country code resolution for PPP pricing
  - Endpoint: `http://ip-api.com/json/{ip}?fields=countryCode`
  - Usage: Server-side IP geolocation fallback in `app/lib/country.server.ts`
  - Fallback behavior: Silently fails if unavailable
  - No auth required (public API)

**YouTube:**
- YouTube IFrame API - Video embedding and playback
  - SDK: YouTube IFrame API JavaScript library (`https://www.youtube.com/iframe_api`)
  - Implementation: `app/components/youtube-player.tsx`
  - Features: Video playback, progress tracking, autoplay control
  - Client-side only

**Discord:**
- Discord Community (optional integration)
  - Purpose: Community engagement link
  - Configuration: `DISCORD_INVITE_URL` environment variable
  - Implementation: `app/routes/courses.$slug.welcome.tsx`
  - Used in: Course welcome page for enrolled students

**Google Fonts:**
- Google Fonts CDN - Typography
  - Font: Inter (variable font with weights 100-900)
  - URLs:
    - `https://fonts.googleapis.com/css2?family=Inter...`
    - `https://fonts.gstatic.com` (preconnect)
  - Usage: `app/root.tsx`

**Cloudflare:**
- Cloudflare Headers (optional)
  - Header: `CF-IPCountry` - Country code from Cloudflare
  - Fallback source for geolocation in `app/lib/country.server.ts`
  - Tier 1 priority (before ip-api.com fallback)

## Data Flow & Integration Patterns

**Geolocation Flow:**
1. Request includes headers: `X-Forwarded-For`, `CF-IPCountry`, or session dev country
2. `app/lib/country.server.ts` resolves in order:
   - Dev session override (if set)
   - Cloudflare header (fastest)
   - IP-API.com fallback (if IP available)
3. Country code stored temporarily for PPP pricing tier calculation

**Video Tracking Flow:**
1. YouTube Player loads video from lesson data
2. Client-side player emits tracking events (play, pause, progress, ended)
3. Events sent to `/api/video-tracking` endpoint (best-effort, silent failures)
4. Server stores events in `videoWatchEvents` table
5. Progress tracked for completion metrics

**Rating System Flow:**
1. User submits rating (1-5) for enrolled course
2. POST to `/api/course-rating` requires authentication
3. Server validates enrollment via `isUserEnrolled()`
4. Upserts rating record in `courseRatings` table (one per user+course)
